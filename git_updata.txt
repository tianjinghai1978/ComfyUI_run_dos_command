github 上传方法
以下是使用 Git 命令将本地项目上传到 GitHub 的详细步骤：

1. 准备工作
安装 Git 并配置用户信息：
bash
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"


在 GitHub 上创建一个新仓库（Repository），获取仓库地址（HTTPS 或 SSH 格式）

2. 上传步骤
初始化本地仓库（如果项目尚未使用 Git）
bash
# 进入项目文件夹
cd 你的项目路径

# 初始化 Git 仓库
git init


添加文件并提交
bash
# 添加所有文件到暂存区
git add .

# 提交到本地仓库，填写提交信息
git commit -m "首次提交：初始化项目"
关联远程仓库并推送
bash
# 关联 GitHub 仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

# 推送本地分支到远程仓库（首次推送需要 -u 参数）
git push -u origin main

3. 后续更新推送
当项目有更新时，只需执行：
bash
git add .
git commit -m "描述本次修改内容"
git push
注意事项
如果远程仓库非空（有 README 等文件），需先拉取再推送：
bash
git pull origin main --allow-unrelated-histories

推荐使用 SSH 连接方式，避免每次输入密码，需要提前配置 SSH 密钥
如果使用 GitHub Desktop 等图形化工具，可以简化上述命令操作，通过界面按钮完成提交和推送。
